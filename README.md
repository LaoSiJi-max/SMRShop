# SMRShop

[中文](README.md) | [日本語](README_ja.md)

## 系统结构

项目采用传统的 Java Web 分层结构：

```text
浏览器
  │
  ▼
JSP 页面
  │
  ▼
Struts Action
  │
  ▼
Service 业务逻辑
  │
  ▼
DAO 数据访问
  │
  ▼
Hibernate
  │
  ▼
MySQL
```

各层的主要职责如下：

* **JSP**
  负责商城前台和后台管理页面的显示。

* **Action**
  接收页面请求，调用业务逻辑，并控制页面跳转。

* **Service**
  处理用户登录、购物车、订单、图书和管理员管理等业务。

* **DAO**
  负责数据库的增删改查操作。

* **Hibernate**
  负责 Java 对象与数据库表之间的映射和持久化。

* **Spring**
  负责对象管理、依赖注入和事务配置。

## 架构上的尝试

这个项目中，我主要尝试了以下设计：

### 接口与实现分离

Action、Service和DAO都分别定义了接口和实现类，例如：

```text
BookAction
└── BookActionImpl

BookService
└── BookServiceImpl

BookDao
└── BookDaoImpl
```

这样可以减少各层之间对具体实现的直接依赖。

### 通用DAO

项目定义了泛型形式的 `BaseDao<T>` 和 `BaseDaoImpl<T>`，集中实现常见的数据库操作，再由图书、用户、管理员和订单DAO继承或扩展。

```text
BaseDao<T>
└── BaseDaoImpl<T>
    ├── BookDaoImpl
    ├── UserDaoImpl
    ├── AdminDaoImpl
    └── OrderDaoImpl
```

### 依赖注入

DAO、Service和Action之间的依赖关系统一交给Spring配置和注入，而不是在业务代码中直接创建对象。

### 事务管理

项目通过Spring和Hibernate配置事务，对DAO层的数据操作进行统一管理。

### 枚举状态

用户、管理员和订单的状态使用枚举表示，例如：

* 用户状态
* 管理员状态
* 管理员等级
* 订单状态

### 分页处理

图书列表使用独立的 `Paging` 工具类计算当前页、起始位置、结束位置和总页数。

## 购物和订单流程

```text
浏览图书
   │
   ▼
查看图书详情
   │
   ▼
加入购物车
   │
   ▼
查看购物车和总价
   │
   ▼
提交购物车
   │
   ▼
生成订单
   │
   ▼
查看个人订单
```

用户提交购物车后，系统会根据购物车中的图书生成订单，并清空原购物车内容。管理员可以在后台查看订单，并修改订单状态。
