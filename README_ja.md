# SMRShop

[中文](README.md) | [日本語](README_ja.md)

SSH（Spring、Struts、Hibernate）フレームワークを使用して開発した書籍通販システムです。

## プロジェクト紹介

これは2016年の期末課題として制作したプロジェクトで、名前は「謎の覆面人ショップ」です。なぜこの名前になったのかについては、友人の一人に感謝しなければなりません。（￣▽￣）

当時はJava Webを学び始めてから約1年が経っており、どうすればプロジェクトの構成を分かりやすくし、開発もしやすくできるのかをずっと考えていました。このプロジェクトは、その時点で「少しだけ分かってきた」と感じられた成果の一つです。

フロントエンドにはオープンソースのテンプレートを使用し、主にバックエンド機能とプロジェクト構成の実装に力を入れました。

当時の実行画面はもう残っていませんし、今からこのプロジェクトの復活の儀式を行うつもりもありません。∑( 口 ||

このリポジトリでは、初期の学習過程の記録として、主にソースコードとプロジェクト構成を保存しています。

## 主な機能

### フロント画面

* 書籍一覧の表示
* キーワードによる書籍検索
* 書籍詳細情報の表示
* ユーザー登録・ログイン
* ユーザー状態の確認
* 書籍をカートに追加
* カートから書籍を削除
* カート内商品の合計金額計算
* カート内容を送信して注文を作成
* 自分の注文履歴の確認

### 管理画面

* 管理者ログイン
* 書籍情報の一覧表示・検索
* 書籍の追加・変更
* 書籍の削除
* ユーザー情報の一覧表示・検索
* ユーザーアカウントの有効化・無効化
* ユーザーの削除
* 管理者アカウントの作成・削除
* 管理者アカウントの有効化・無効化
* 全注文の確認
* 注文状態の変更

## システム構成

このプロジェクトは、従来型のJava Webのレイヤー構成を採用しています。

```text
ブラウザ
  │
  ▼
JSP画面
  │
  ▼
Struts Action
  │
  ▼
Service ビジネスロジック
  │
  ▼
DAO データアクセス
  │
  ▼
Hibernate
  │
  ▼
MySQL
```

各レイヤーの主な役割は以下のとおりです。

* **JSP**
  ECサイトのフロント画面と管理画面の表示を担当します。

* **Action**
  画面からのリクエストを受け取り、ビジネスロジックを呼び出し、画面遷移を制御します。

* **Service**
  ユーザーログイン、カート、注文、書籍、管理者管理などのビジネスロジックを処理します。

* **DAO**
  データベースに対する登録、取得、変更、削除を担当します。

* **Hibernate**
  Javaオブジェクトとデータベーステーブルのマッピング、およびデータの永続化を担当します。

* **Spring**
  オブジェクト管理、依存性注入、トランザクション設定を担当します。

## アーキテクチャ上の試み

このプロジェクトでは、主に以下の設計を試しました。

### インターフェースと実装クラスの分離

Action、Service、DAOについて、それぞれインターフェースと実装クラスを分けています。

```text
BookAction
└── BookActionImpl

BookService
└── BookServiceImpl

BookDao
└── BookDaoImpl
```

これにより、各レイヤーが具体的な実装クラスへ直接依存しすぎない構成を目指しました。

### 共通DAO

ジェネリクスを使用した `BaseDao<T>` と `BaseDaoImpl<T>` を定義し、一般的なデータベース操作を共通化しています。

書籍、ユーザー、管理者、注文に対応する各DAOは、この共通DAOを継承または拡張しています。

```text
BaseDao<T>
└── BaseDaoImpl<T>
    ├── BookDaoImpl
    ├── UserDaoImpl
    ├── AdminDaoImpl
    └── OrderDaoImpl
```

### 依存性注入

DAO、Service、Actionの依存関係はSpringの設定によって注入し、ビジネスコード内で直接オブジェクトを生成しない構成にしました。

### トランザクション管理

SpringとHibernateを使用してトランザクションを設定し、DAO層のデータ操作を一括して管理しています。

### Enumによる状態管理

ユーザー、管理者、注文に関する状態はEnumで管理しています。

* ユーザー状態
* 管理者状態
* 管理者権限レベル
* 注文状態

### ページング処理

書籍一覧では、独立した `Paging` クラスを使用し、現在ページ、開始位置、終了位置、総ページ数などを計算しています。

## 購入・注文処理の流れ

```text
書籍を閲覧
   │
   ▼
書籍詳細を確認
   │
   ▼
カートに追加
   │
   ▼
カート内容と合計金額を確認
   │
   ▼
カートを送信
   │
   ▼
注文を作成
   │
   ▼
自分の注文履歴を確認
```

ユーザーがカート内容を送信すると、カート内の書籍をもとに注文が作成され、その後カートの内容が空になります。

管理者は管理画面から注文を確認し、注文状態を変更できます。
